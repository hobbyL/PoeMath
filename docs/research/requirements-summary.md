### 2.3 数据来源

| 数据源 | 规模 | 内容 | 许可证 | 用途 |
|--------|------|------|--------|------|
| **yxcs/poems-db** | 22万首 | 注释、赏析、作者简介、分类标签 | 自由使用 | **主要数据源** |
| **byj233/ChinesePoetryLibrary** | 80万首 | 译文、赏析、两版注释、典故 | MPL 2.0 | 补充数据 |
| **chinese-poetry/chinese-poetry** | 30万+ | 仅原文 | MIT | 原文参考 |
| **VMIJUNV/chinese-poetry-and-prose** | 84万篇 | 译文、赏析、热度 | 学习交流 | 热度排序参考 |

数据整理流程：
1. 从 yxcs/poems-db 提取含注释赏析的完整数据
2. 按三层目录筛选（课标→经典→主题）
3. 用 lpinyin 批量生成拼音
4. 核心层130首逐首人工校验
5. 打包为本地 JSON/SQLite 内嵌到 App

预估数据体积：~2.6MB（不影响App大小）

## 三、口算模块

### 3.1 题型覆盖——按年级

| 年级 | 题型范围 | 关键约束 |
|------|---------|---------|
| 一年级上 | 10以内加减法 | 结果≥0，无进退位 |
| 一年级下 | 20以内加减法 | 可进退位，结果≥0 |
| 二年级上 | 100以内加减法、乘法口诀 | 两位数±一/两位数 |
| 二年级下 | 万以内加减、有余数除法 | 表内除法，余数<除数 |
| 三年级上 | 多位数乘一位数、万以内加减 | 竖式计算 |
| 三年级下 | 两位数乘两位数、除法竖式 | 三步混合运算 |
| 四年级上 | 大数运算、三步混合运算 | 运算律(交换/结合/分配) |
| 四年级下 | 小数加减法 | 小数点对齐 |
| 五年级上 | 小数乘除法 | 积/商的小数位数 |
| 五年级下 | 分数加减法 | 通分、最简分数 |
| 六年级上 | 分数乘除法、百分数 | 倒数、分数÷分数 |
| 六年级下 | 比例、正负数 | 比例尺、负数运算 |

### 3.2 出题引擎设计

核心算法：**参数化随机生成 + 约束验证**（纯Dart实现，~300-500行）

```
lib/math_engine/
├── models/
│   ├── math_problem.dart      // 题目数据模型
│   └── grade_config.dart      // 年级配置模型
├── generators/
│   ├── base_generator.dart    // 生成器基类
│   ├── arithmetic_gen.dart    // 加减乘除生成器
│   ├── vertical_gen.dart      // 竖式题生成器
│   ├── fraction_gen.dart      // 分数题生成器
│   ├── equation_gen.dart      // 简易方程生成器
│   ├── compare_gen.dart       // 比大小生成器
│   └── chain_gen.dart         // 连续运算生成器
├── validators/
│   ├── constraint_checker.dart // 约束验证器
│   └── difficulty_scorer.dart  // 难度评估器
├── presets/
│   └── grade_presets.dart     // 1-6年级题型预设配置
└── math_engine.dart           // 统一入口
```

参考开源项目（算法参考，不直接移植）：
- **math-whiz**（nilIfNull）—— 人教版章节级精确对齐，MIT
- **smartmatch**（zgjff）—— 1-6年级全覆盖，MIT
- **little-math**（zx06）—— 代码结构最清晰，TypeScript，MIT
- **PrimaryOralMathPack**（bosichong）—— NPM包，功能丰富，Apache 2.0

### 3.3 出题模式

| 模式 | 说明 | 优先级 |
|------|------|--------|
| 求结果 | 25 + 38 = ? | P0 |
| 求未知项(填空) | 25 + ? = 63 | P0 |
| 比大小 | 25 + 38 ○ 70 | P1 |
| 竖式计算 | 展示竖式格式 | P1 |
| 连续运算 | 3 + 5 - 2 = ? | P1 |
| 带括号运算 | (3 + 5) × 2 = ? | P1 |

### 3.4 错题系统

| 功能 | 实现方式 | 优先级 | 是否需要联网 |
|------|---------|--------|-------------|
| 错题记录 | 本地数据库存储每次答错的题目 | P0 | ❌ |
| 错题重练 | 从错题本中抽题重新练习 | P0 | ❌ |
| 计算步骤讲解 | 程序化生成解题步骤（基于规则） | P0 | ❌ |
| 错因分析 | 基于错误模式匹配（进位/退位/口诀等） | P1 | ❌ |
| 间隔复习 | 艾宾浩斯曲线调度错题复习 | P1 | ❌ |
| 同类题推荐 | 生成同类型同难度的新题 | P1 | ❌ |
| AI深度讲解 | 大模型API生成详细解释 | P3 | ✅ |

#### 错因分析规则表（程序化，无需AI）

| 错误类型 | 检测方法 | 讲解模板 |
|---------|---------|---------|
| 进位遗漏 | 正确答案比学生答案大10的倍数 | "个位相加超过10，需要向十位进1" |
| 退位遗漏 | 学生答案比正确答案大10的倍数 | "个位不够减，需要从十位借1当10" |
| 乘法口诀错误 | 对比乘法表 | 展示正确口诀 |
| 运算顺序错误 | 检查混合运算 | "先算乘除，后算加减" |
| 余数错误 | 余数≥除数 | "余数必须比除数小" |
| 小数点位置错误 | 小数位数不对 | "小数点要对齐" |

### 3.5 计算公式知识库

**本质：静态知识库，约60-80条公式，按年级分类展示**

#### 公式分类

| 类别 | 数量 | 首次出现年级 | 示例 |
|------|------|------------|------|
| 几何周长 | 3条 | 3年级 | 长方形周长 C=(a+b)×2 |
| 几何面积 | 6条 | 3年级 | 三角形面积 S=ah÷2 |
| 几何体积 | 4条 | 5年级 | 圆柱体积 V=πr²h |
| 单位换算 | ~25条 | 1年级 | 长度/面积/体积/质量/时间 |
| 数量关系 | ~8条 | 3年级 | 速度×时间=路程 |
| 运算定律 | ~6条 | 4年级 | 乘法分配律 (a+b)×c=a×c+b×c |
| 分数/小数规则 | ~8条 | 5年级 | 异分母通分、小数除法 |
| 比例 | ~4条 | 6年级 | 正比例 y/x=k |

#### 数据结构

```json
{
  "id": "formula_001",
  "category": "几何面积",
  "name": "三角形面积",
  "formula_text": "S = ah ÷ 2",
  "formula_latex": "S = \\frac{ah}{2}",
  "grade": 4,
  "params": [
    {"symbol": "S", "meaning": "面积"},
    {"symbol": "a", "meaning": "底"},
    {"symbol": "h", "meaning": "高"}
  ],
  "memory_tip": "底乘高除以2",
  "example": "底=6cm，高=4cm → S=6×4÷2=12cm²",
  "related_formulas": ["formula_002", "formula_003"]
}
```

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 分类浏览 | P0 | 按类别（几何/单位换算/数量关系）和年级浏览 |

### 4.2 儿童UX原则

| 原则 | 具体要求 |
|------|---------|
| 大按钮 | 可点击区域 ≥ 44pt |
| 浅层级 | 导航层级 ≤ 2层 |
| 拟物化 | 使用具象图标（书本、铅笔、星星） |
| 正向鼓励 | 答错时鼓励"再试一次"，不使用"×"或红色 |
| 短时长 | 每次练习控制在10-15分钟 |
| 零广告 | 完全无广告 |
| 大字体 | 正文 ≥ 16sp，诗词内容 ≥ 20sp |
| 护眼模式 | 支持暖色调/深色模式 |

## 九、UI设计规范

### 9.1 设计风格定位

**双主题方案**——用户可切换：

| 主题 | 定位 | 场景 |
|------|------|------|
| 🎋 **国风水墨** | 宣纸色 + 墨绿 + 朱砂 + serif 字体 | 诗词模块主视觉 |
| 🎨 **童趣马卡龙** | 高饱和浅色 + 圆润字体 + IP形象 | 口算模块主视觉 |

设计原则：**诗词区域偏古典雅致，口算区域偏活泼童趣**，通过配色和字体在同一App中做出场景区分。

#### 9.2.2 童趣马卡龙色板（口算场景）


### 9.8 无障碍设计

| 要求 | 具体实现 |
|------|---------|

| 深色模式 | 支持系统深色模式 |

### 9.9 反面清单（避免踩坑）

参考 Smashing Magazine《Designing For Children》：

- ❌ **不使用底部隐藏按钮** — 儿童容易误触
- ❌ **不使用"×"错误标记** — 有惩罚感，改用💧或"再试试"
- ❌ **不出现广告/内购** — 家长最反感
- ❌ **不使用小于44pt的按钮**
- ❌ **不使用小于16sp的正文**
- ❌ **不使用纯黑纯红** — 视觉压迫感
- ❌ **不用无反馈的操作** — 儿童期待每个操作都有响应
- ❌ **不用超过2层的导航** — 儿童容易迷路
- ❌ **不用需要精确操作的手势** — 儿童手指控制力弱
- ❌ **不用惩罚性文案** — 用"再想想""加油"代替"错了"
